### A11y вЂ” Pressables without role/label across 7 modals
Standard pattern: close-X buttons, dismiss links, option cards, rating
toggles, share actions, send buttons all rendered as bare `<Pressable>`
without `accessibilityRole`/`accessibilityLabel`. Invisible to VoiceOver
and to `mobilecli dump ui` filtered by `type === 'Button'`.

Files patched:
- `(modals)/disclaimer.tsx` вЂ” close X + "Not now"
- `(modals)/checkin.tsx` вЂ” back arrow + 3 sugar option cards
  (`role="radio"` + `state.selected` + label) + 5 mood tiles
- `(modals)/craving-log.tsx` вЂ” close X + 5 intensity stones (radio) + 6
  trigger chips (checkbox) + 2 outcome cards (radio) + notes TextInput label
- `(modals)/paywall-contextual.tsx` вЂ” close X + 2 price toggles + "Not now"
- `(modals)/share-card.tsx` вЂ” close X + 3 share actions
- `(modals)/rate-app.tsx` вЂ” close X + 5 rating stones + "Maybe later"
- `(modals)/sos.tsx` вЂ” back arrow (chat + blocked variant) + send button
  + TextInput label/hint + blocked-screen "Not now"

After fix, dump now exposes every interactive element with a descriptive
label. Example for paywall:
