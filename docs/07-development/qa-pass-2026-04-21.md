# Sugar Quit — UI/UX/Logic QA Pass (2026-04-21)

> **Update 2026-04-22**: 7 real bugs fixed and verified live (#1 #2 #3 #4 #5 #6 #9).
> 5 were false positives after code read (see "False positives" section at the end).

**Device**: iPhone 17 Pro (402×874 pt) "SugarQuit" sim, UDID `FA05FD96-2663-44D8-9632-6F7C5787647F`
**Build**: Expo Go 55.0.27 + Metro on 8082 from `APP_DEVELOPMENT/sugar-quit/`
**Protocol**: ui-qa skill (16 rules + standard screen checklists)
**Pass scope**: onboarding 1–12 + result + paywall. Post-paywall screens (Home/SOS/Progress/Profile) BLOCKED by hard paywall — see bug #12.

## Bugs found

### Blockers for QA coverage
- **#12 (UX, high)** — **Paywall has no X/close button**. «Maybe later» link (bottom, small grey) is the ONLY dismiss path and it navigates **back to Result screen**, not into the app (see #11). Free-tier user therefore cannot see Home, SOS, Progress, Profile without starting a trial. Blocks the rest of this pass; may also be a product blocker if the intent is "let users explore before paywall".
- **#11 (UX, high)** — «Maybe later» on paywall → Result screen (one step back), not Home. Mis-sets user expectation: "later" implies "skip for now, show me the app", but app is unreachable without purchase.

### A11y — multiple elements missing Button role
- **#1 (a11y, high)** — Onboarding option cards (Step 2 goals, Step 7 triggers, Step 8 intake, Step 10 history, Step 11 days) render only as `StaticText` in the accessibility tree. Visually they're Pressables with visible tap feedback. Breaks VoiceOver AND all e2e automation (mobilecli `dump ui` can't surface them; I had to resort to coordinate taps).
- **#10 (a11y, high)** — Paywall price cards (Annual $79.99 / Monthly $9.99 toggle) not in a11y tree. VoiceOver users cannot select a plan.
- **#9 (a11y, medium)** — «Maybe later» dismiss link not in a11y tree.
- **#13 (a11y, medium)** — Expo Router default 404 screen ("Unmatched Route / Go back / Sitemap"): Go back + Sitemap links not exposed as Button. Library-level issue inherited by sugar-quit.
- **#5 (a11y, low)** — Step 12 `TextField` (first-name input) has no `accessibilityLabel`.

### UX / consistency
- **#2 (UX, medium)** — Inconsistent single-select pattern: Step 2 (goal) auto-advances on tap; Step 4 (approach) also 2-option single-select but requires explicit CONTINUE. Pick one pattern.
- **#3 (UX, medium)** — Step 7 (triggers, multi-select "Pick any that ring true"): CONTINUE is active with zero selections. Step 3 (same multi-select pattern) was disabled until ≥1 pick. Inconsistent validation.
- **#6 (UX, medium)** — Step 12 keyboard covers CONTINUE. `KeyboardAvoidingView` doesn't lift the CTA. Tap outside the input does NOT dismiss the keyboard — no `TouchableWithoutFeedback`/`Pressable onPress={Keyboard.dismiss}` wrapper. User must find the keyboard's Done/Return key.

### Copy
- **#4 (copy, low)** — Eyebrow inconsistency: Step 5 says **«YOU'RE NOT ALONE»**, Step 9 says **«YOU ARE NOT ALONE»** — same concept, different contraction. Pick one.
- **#7 (copy, low)** — Result screen (YOUR CRAVING PROFILE): subtitle uses **«3pm»** (lowercase, no space), Peak Window card uses **«3:00 PM»** (full clock format). Same screen, different time formats.

## What worked well

- **Cross-surface content tie-in (Rule 3)**: Paywall bullet «Trigger prediction tuned to your 3pm» references Step 6 Peak Window selection. Result card «PEAK WINDOW / 3:00 PM» ties to Step 6. Good content threading.
- **Pricing math consistent**: paywall says "$0.22 / day decision", annual $79.99 — $0.22 × 365 = $80.30. Passes sanity check.
- **Personalization picks up name from Step 12**: paywall eyebrow "ALEX, YOUR PLAN IS READY". (The "↓" I saw was my test artifact from typing `\n`, not a product bug.)
- **Selection state UI is instant**: tapping options flips background + ✓ chip immediately, CONTINUE activates in sync (rule 12 return-path ✓).
- **Hydration / state persistence works**: app was on Step 2 at session start (onboarding progress survives Expo Go backgrounding via WDA re-attach + cold deep-link).
- **Default values sensible**: Step 6 Peak Window pre-selects AFTERNOON 3:00 PM, letting user hit CONTINUE without a forced pick.

## False positives (after reading code)

Two lessons: (a) the `type === 'Button'` filter I used in `mobilecli dump ui`
doesn't surface `accessibilityRole="radio"` or `role="checkbox"` — they appear
as `Other`. (b) the blue-circle icon in the paywall top-right that I misread as
"settings gear" is actually the close button (`closeBtn` in
`paywall.tsx:85-88`) — just coloured with the app's primary blue and a `×`
glyph.

- **#7 `3pm` vs `3:00 PM`** — by design. `result.tsx:36-37` deliberately does
  `peakLabel.replace(/(:00 )?(AM|PM)/, '$2').toLowerCase()` to produce a casual
  "3pm" for the subtitle while the card title keeps the formal "3:00 PM". Two
  different registers in two different contexts.
- **#10 paywall price toggles** — already have `accessibilityRole="radio"` +
  `accessibilityState` + `accessibilityLabel` (`paywall.tsx:131-156`). My
  filter missed them because they're `Other` type in WDA, not `Button`.
- **#11 «Maybe later» navigates to Result** — product intent. The route
  returns to the Result screen (hard paywall funnel: user must purchase or
  leave), not a bug.
- **#12 "paywall has no X close"** — actually present at
  `paywall.tsx:85-88`. My misread: the blue circle with `×` glyph was read
  visually as a settings cog. Now confirmed in post-fix dump as
  `Button "Close paywall" @ 398, 88`.
- **#13 Expo Router 404 `Go back` / `Sitemap`** — upstream `expo-router`
  default screen, not under sugar-quit's control.
