# App Store Screenshot Captions — Unsweet (en-US)

7 frames × hero (≤30 chars OCR-indexed since June 2025) + supporting sub (≤60 chars).

**Critical**: First 3 frames = ~80% conversion (visual playbook §4). Order frames by impact, not by app navigation order.

**OCR indexing**: every hero caption acts as **Tier 4 keyword carrier**. We treat captions as a 4th hidden keyword field — they MUST contain Tier 4 keywords from aso-pack.

## Total OCR keyword surface (Tier 4)

Across 7 captions we expose these phrases to Apple's OCR indexing:

```
AI, Coach, crave, cravings, identity, willpower, 
detox, plan, neuroscience, peak, hour, SOS, 
sugar, 90 days, 60 seconds
```

This adds ~14 keyword tokens to our combinatorial pool that aren't in name + subtitle + keywords field. Effectively a 4th keyword field, free.

## Design rules per visual playbook

- Font: clean sans-serif (Inter / SF Pro / Helvetica Now)
- Hero caption: 80-100pt, weight 700 (bold)
- Supporting sub: 36-44pt, weight 400 (regular)
- Color: dark text on light background OR white text on cool-blue/indigo background
- NO red/orange (sugar craving trigger, per visual playbook §12)
- Padding around caption: 80px top/bottom from frame edge

## Localization notes

For non-English locales, **caption text must be re-researched per locale** — direct translation loses keyword targeting. Section 7 textual playbook: "Keywords are NEVER translated literally."
